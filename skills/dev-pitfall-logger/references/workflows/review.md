# 定期回顾 - 详细工作流

## 核心原则

**轻量级、非侵入式、用户主动触发。**

---

## 触发方式

### 方式一：用户主动回顾（主要）

**触发词**:
- "帮我回顾一下这个月的踩坑记录"
- "本周有哪些值得注意的问题？"
- "小程序项目遇到过哪些坑？"
- "总结一下最近的踩坑"

**执行**:
1. 确定时间范围（本周/本月/本项目/全时间）
2. 读取对应时间范围的记录
3. 生成总结报告

### 方式二：创建记录时顺便提醒（可选）

**提醒时机**: 创建记录后，自然附加提醒

**提醒格式**:
```markdown
✅ 已创建踩坑记录: pit-20260224-001

📊 本月已记录 12 条踩坑

💡 要回顾一下本月的其他记录吗？
   (回复"回顾"查看本月总结，或忽略继续工作)
```

**不过度提醒**:
- 每创建5条记录后提醒一次
- 或者用户明确要求"每次都提醒"

### 方式三：项目结束时（高价值）

**触发时机**: 用户提到"项目完成了"/"这个项目结束了"

**执行**: 生成该项目的完整踩坑回顾

## 回顾输出格式

### 标准格式（必须使用）

```markdown
📊 2026年1月踩坑回顾

本月共记录 9 条，按严重程度分布：
- 🔴 Critical: 0条
- 🟠 High: 3条（方向调整、备案驳回、文件同步）
- 🟡 Medium: 2条（产品方向、工具弹窗）
- 🟢 Low: 4条（命名、上传按钮等）

🎯 值得关注的模式：

1. 备案问题反复出现（2次）
   - pit-20260117-001: 小程序备案地区填写
   - pit-20260118-001: 备案被驳回（身份证照片/人脸认证）

   → 建议：一次性拍对，提前了解审核标准

2. 文件同步问题浪费2小时
   - pit-20260120-001: 文件同步问题（软链接不稳定）

   → 建议：避免软链接，直接在正确目录开发

3. 产品方向认知升级
   - pit-20260116-001: 从"我喜欢什么"到"别人需要什么"

   → 认知升级，后续项目保持

📁 按分类统计：
- frontend: 2条
- devops: 4条
- tooling: 2条
- other: 1条

详细记录：
- 按时间: references/logs/2026/2026-01.md
- 按项目: references/logs/by-project/miniprogram.md
```

### 项目回顾格式

```markdown
📊 [项目名称] 踩坑回顾

项目周期: 2026-01-15 ~ 2026-01-25
记录数量: 8条

🎯 核心问题：

1. 备案问题（3次，占比37.5%）
   - 主要难点：审核标准不清晰，反复被驳回
   - 解决：提前阅读审核指南，一次性准备材料

2. 产品方向调整（1次）
   - 问题：从"我喜欢"转向"用户需要"
   - 价值：认知升级，影响后续所有项目

3. 工具配置问题（2次）
   - 微信开发者工具弹窗
   - 文件同步软链接不稳定

✅ 项目经验总结：
- 提前了解平台规则（备案）
- 从用户需求出发，不是个人喜好
- 避免软链接，直接在正确目录工作
```

## 模式识别规则

### 识别值得关注的模式

#### 1. 重复问题
**定义**: 同类问题出现 ≥ 2次

**输出格式**:
```markdown
1. XXX问题反复出现（N次）
   - [列出相关记录]
   → 建议：XXX
```

**判断标准**:
- 同一关键词出现 ≥ 2次
- 同一技术栈/工具出现问题 ≥ 2次
- 同一错误类型出现 ≥ 2次

#### 2. 时间黑洞
**定义**: 单一问题花费大量时间

**输出格式**:
```markdown
2. XXX问题浪费大量时间
   - 花费时间: X小时
   → 建议：XXX
```

**判断标准**:
- 修复时间 > 1小时
- 尝试次数 > 3次

#### 3. 认知升级
**定义**: 改变了思维方式或做法

**输出格式**:
```markdown
3. XXX认知升级
   - 从: [旧做法/旧思维]
   - 到: [新做法/新思维]
   → 后续项目保持
```

**判断标准**:
- 明确提到"认知升级"/"思维转变"
- 影响后续项目决策
- 从根因分析中提取出原则性经验

## 回顾时机建议

### 推荐回顾时机

| 时机 | 价值 | 频率 |
|------|------|------|
| 每周回顾 | 低 | 可选（周末快速浏览） |
| 每月回顾 | 中 | 推荐（月末总结） |
| 项目结束 | 高 | 强烈推荐（完整回顾） |
| 遇到相似问题 | 高 | 自动提醒相关历史记录 |

### 不推荐过度回顾

- ❌ 不定时推送（避免打断工作）
- ❌ 不在对话开始时主动汇报（避免噪音）
- ✅ 只在用户主动询问或添加记录时轻量提醒

---

## 回顾输出内容控制

### 输出长度控制

| 回顾类型 | 推荐长度 | 最大长度 |
|---------|---------|---------|
| 每周回顾 | 5-10行 | 20行 |
| 每月回顾 | 20-30行 | 50行 |
| 项目回顾 | 30-50行 | 100行 |

### 内容优先级

**必须包含**:
1. 统计数据（数量、分类分布）
2. 值得关注的模式（重复问题、时间黑洞）
3. 改进建议

**可选包含**:
1. 详细记录列表（链接到完整记录）
2. 图表（如果数据量足够大）
3. 趋势分析（如果有足够历史数据）
