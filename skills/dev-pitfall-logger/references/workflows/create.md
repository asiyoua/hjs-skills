# 创建记录 - 详细工作流

## 触发条件
用户说以下任一内容时触发：
- "记录/记一下/这个坑"
- "帮我记录一下"
- 明确表示要创建踩坑记录

---

## 第一步：加载模板

**模板位置**: `references/templates/pitfall_template.md`

**加载时机**: 确定任务类型后立即加载

---

## 第二步：收集信息

### 必填字段（必须有值才能创建）

| 字段 | 说明 | 验证标准 |
|------|------|---------|
| date | YYYY-MM-DD | 必须是今天或过去日期 |
| category | frontend|backend|database|devops|tooling|other | 必须是6选1 |
| tech | 技术栈数组 | 至少1个，如 [Node.js] |
| severity | low|medium|high|critical | 必须是4选1 |
| keywords | 关键词数组 | 3-5个，可搜索术语 |

### 内容字段（必须填写）

| 字段 | 最低要求 | 推荐标准 |
|------|---------|---------|
| 错误现象 | 1句话描述 | 详细症状+错误信息 |
| 解决方案 | 1个步骤 | 多个方案+步骤说明 |
| 根因分析 | 可选 | 强烈推荐，说明"为什么" |
| 预防措施 | 可选 | 推荐填写，说明"如何避免" |
| 上下文环境 | 项目名 | 版本+环境+代码位置 |

---

## 第三步：生成记录ID

**格式**: `pit-YYYYMMDD-NNN`

**生成规则**:
1. YYYYMMDD = 今天的日期
2. NNN = 当天的序号（001, 002, 003...）
3. 查询当月文件，确定下一个序号

**示例**:
```
今天是 2026-02-24
当月已有记录: pit-20260224-001, pit-20260224-002
新记录ID: pit-20260224-003
```

---

## 第四步：保存到主文件

**主文件位置**: `references/logs/YYYY/YYYY-MM.md`

**保存规则**:
1. 文件不存在 → 创建文件
2. 文件存在 → 追加到末尾
3. 确保格式正确（使用模板格式）

**记录格式**（必须遵守）:
```markdown
## pit-YYYYMMDD-NNN: [标题]

**日期:** YYYY-MM-DD
**分类:** frontend|backend|database|devops|tooling|other
**技术:** [Tech1, Tech2]
**严重程度:** low|medium|high|critical
**关键词:** [keyword1, keyword2, keyword3]

### 错误现象
[描述问题]

### 解决方案
**方案一：[方案名称]**
[步骤化解决过程]

**方案二：[方案名称]**（可选）
[替代方案]

### 根因分析
[为什么发生]

### 预防措施
[如何避免]

### 上下文环境
- **项目**: [项目名]
- **版本**: [版本号]
- **环境**: [开发/生产环境]
- **相关文件**: [文件路径或链接]
```

---

## 第五步：更新索引

### 索引更新规则（必须遵守）

**原则**: 索引只包含链接，不包含数据

**更新顺序**:
1. **分类索引**: `references/logs/by-category/{category}.md`
2. **项目索引**: `references/logs/by-project/{project}.md`
3. **主索引**: `references/index.md`（仅 high/critical）

### 分类索引格式
```markdown
# {Category} Pitfalls

## 2026-02
- **[pit-20260224-001](../../2026/2026-02.md#pit-20260224-001):** [标题]
  - Category: {category} | Severity: {severity}
```

### 项目索引格式
```markdown
# {Project} Pitfalls

## 2026-02
- **[pit-20260224-001](../../2026/2026-02.md#pit-20260224-001):** [标题]
  - Date: 2026-02-24 | Severity: {severity}
```

---

## 第六步：确认用户

**确认信息**:
```markdown
✅ 已创建踩坑记录

📝 ID: pit-20260224-001
📍 位置: references/logs/2026/2026-02.md
🏷️  分类: frontend | 严重程度: medium
🔍 已索引: by-category/frontend.md, by-project/miniprogram.md

预览：
⚡ 问题：[1句话概括]
💡 解决：[关键步骤]
📌 关键：[最重要1点]
```

---

## 特殊情况处理

### 信息不完整
```
我需要更多信息来创建记录：

✅ 已有：
- 分类: frontend
- 技术: [Next.js]

❌ 还需要：
1. 具体的错误现象是什么？
2. 你是怎么解决的？

请提供以上信息，我会帮你完成记录。
```

### 查重逻辑
**创建前必须检查是否有重复记录**:

1. 用关键词搜索现有记录
2. 如果找到相似度 > 80% 的记录
3. 提示用户：
   ```
   ⚠️ 发现相似记录：
   [pit-20260115-003] Next.js Image 组件在生产环境显示问题

   是同一问题吗？
   - 是 → 更新现有记录
   - 否 → 创建新记录
   ```
